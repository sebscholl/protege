# Project: Operation Gigglesnort

Generated by Protege AI on exploration of the repository.

## What Is This Project?

Protege is an **email-native AI agent framework** built for local-first, self-hosted deployment. The core idea is simple but powerful: instead of a chat UI or proprietary app, you interact with an AI persona entirely through **email**. Send an email, get a reply. The AI lives on your machine.

## Architecture

The project is organized into three main areas:

### 1. Engine (Core Runtime)
- **Gateway** — Receives inbound SMTP messages, routes them to the right AI persona, runs the AI harness (LLM inference + tool execution loop), and sends replies. Supports both direct SMTP and relay-tunneled modes.
- **Harness** — The inference loop: assembles context from thread history, calls the LLM provider, dispatches tool calls, and routes the final response back.
- **Scheduler** — A cron-based task runner that enables proactive/scheduled agent behaviors ("responsibilities").
- **CLI** — The `protege` command: `setup`, `init`, `gateway start/stop`, `status`, `doctor`, `logs`, `chat`, `relay`, etc.

### 2. Relay (Optional Public Access)
A standalone WebSocket + SMTP bridge that lets Protege instances behind firewalls (e.g., where port 25 is blocked) receive email from the public internet. Routing is based on the persona's public key. Can be self-hosted.

### 3. Tools / Extensions
First-party tools available to the AI:
- `send_email`, `read_file`, `write_file`, `edit_file`, `glob`, `search`, `shell`, `web_fetch`, `web_search`

Tools are registered via an extension manifest and can be configured per-persona via deep-merge overrides.

## Key Design Decisions
- **Personas** are the unit of identity — each has a keypair, an email address, and its own scoped memory/config.
- **No cloud lock-in** — all agent logic, memory, and config live on the user's machine.
- **Relay is opt-in and open** — the relay source is in the repo; you can self-host it.
- **File tools are unsandboxed in v1** (ADR-0025) — simple and reliable semantics prioritized over sandboxing complexity.
- **Tool failures are recoverable** (ADR-0029) — errors are returned to the model as structured results so it can correct and retry within a configurable turn budget.

**Fun Note**: If Operation Gigglesnort ever goes rogue, just unplug the machine and blame it on a 'giggle glitch'—we've programmed it to laugh off any errors! 😄

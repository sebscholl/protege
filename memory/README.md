# Memory

Extension Surface: No

This directory stores runtime state generated by the agent.

It includes SQLite data, attachment files, and logs. Treat contents as runtime artifacts, not source code.

## Memory Model

1. `temporal.db` is temporal memory.
2. `active.md` is active memory.
3. Runtime data is persona-scoped under `memory/{persona_id}/...`.

Temporal memory:

1. Stores chronological history and event ordering.
2. Supports sequence reconstruction and thread/event recall.
3. Backed by SQLite for query/search operations.

Active memory:

1. Stores short-horizon working context.
2. Represents small, quickly accessible state for immediate tasks.
3. Stored as markdown for transparent, human-editable inspection.

Namespace layout:

1. `memory/{persona_id}/temporal.db`
2. `memory/{persona_id}/active.md`
3. `memory/{persona_id}/attachments/`
4. `memory/{persona_id}/logs/`
